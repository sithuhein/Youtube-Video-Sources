# AL Defensive Principles & Techniques

## Input Validation
Checking all incoming data (from users, files, networks) to ensure it meets expected criteria before processing.

## Assertions
Using assert statements to check for conditions that should always be true, helping catch logical errors during development.

## Error Handling
Using try...catch/except blocks to gracefully manage exceptions and unexpected events, logging errors and informing users instead of crashing.

## Boundary Checks
Validating array/list indices and string lengths to prevent buffer overflows.

## Principle of Least Privilege
Granting only necessary permissions to reduce potential damage from security flaws.

## Failsafe Defaults
Designing systems to default to a safe state if an error occurs.
